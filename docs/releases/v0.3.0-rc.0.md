# CallLint v0.3.0-rc.0

> Draft GitHub Release notes. Mark as **Pre-release**. Tag: `v0.3.0-rc.0`.

First release candidate for the stable `0.3.0` line, on top of
[v0.3.0-preview.1](./v0.3.0-preview.1.md). **No scanner-semantics change**: no
detector, verdict, golden expectation, or exit code was altered since preview.1.

An rc is not a feature release — it is a *stable candidate*. Its job is to prove
the release path end-to-end before `0.3.0` claims the default `latest` dist-tag.

```bash
npx calllint@next scan .cursor/mcp.json
```

## What this rc validates

- The release workflow publishes a release candidate to the **`next`** dist-tag
  (not `preview`, not `latest`).
- npm Trusted Publishing (OIDC) build provenance is generated and verifiable
  (`npm audit signatures`).
- The published bundle installs and runs from a clean environment via `npx`.
- The R2.1 corpus gate (`corpus:test` / `corpus:test:r2-final`) is green in the
  release pipeline.
- Public surfaces (README, website, docs) stay internally consistent.

## What shipped since preview.1

Carried in from `main` (these first reach npm in this rc):

- **R2.1 corpus gate** — 30 calibrated cases, 20 of them real-public or
  redacted-real snapshots with per-case origin metadata; dangerous false-SAFE
  = 0; UNKNOWN ratio 10% (target < 15%). Enforced by `corpus:test:r2-final`.
- **SARIF dogfood** —
  [`calllint-demo-risky-mcp`](https://github.com/calllint/calllint-demo-risky-mcp)
  runs CallLint in GitHub Actions; one Code Scanning alert per finding.
- **Website V3** — agent-readable surface (`/llms.txt`, `/agent-instructions.md`,
  `/report-schema.md`, `/security-boundaries.md`) plus corpus and
  release-integrity sections.
- **Release workflow** now routes `*-rc.*` versions to the `next` dist-tag,
  keeping release candidates off the `preview` channel.

## Known limitations

Unchanged from preview.1:

- Static configuration scanner only — does not execute target MCP servers.
- Does not prove runtime safety; a clean run is necessary, not sufficient.
- The R2.1 corpus meets its thresholds but does not yet represent the full MCP
  ecosystem.
- Verdicts are heuristic decision support, not a safety guarantee.
- Pre-1.0 release candidate.

## Note on dist-tags

This rc publishes to **`next`**: install it explicitly with `npx calllint@next`.

- `next` → `0.3.0-rc.0` (this build)
- `preview` → `0.3.0-preview.1`
- `latest` → still `0.3.0-preview.0` (a known drift from the first publish)

The `latest` drift is **not** corrected by this rc. It is fixed at the first
stable release, when `latest` is pointed at `0.3.0` — see
[RELEASE_VERIFICATION.md](../RELEASE_VERIFICATION.md) §1 and
[STABLE_RELEASE_GATE.md](../STABLE_RELEASE_GATE.md).

## Next: 0.3.0 stable

After an RC feedback window (see
