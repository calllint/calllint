# CallLint v0.3.0

> Draft GitHub Release notes. Mark as **latest release** (NOT pre-release).
> Tag: `v0.3.0`.

First **stable** release of CallLint, on top of
[v0.3.0-rc.1](./v0.3.0-rc.1.md). Promotes the validated release candidate to the
default `latest` dist-tag.

**No scanner-semantics change since `0.3.0-rc.1`.** The engine, detectors, verdict
rules, golden expectations, report schema, and exit codes are byte-identical to
rc.1 (`git diff v0.3.0-rc.1 v0.3.0 -- packages/ apps/cli/src/` is empty except a
prose-only corpus README). This release is a promotion plus a dist-tag correction,
not new behaviour.

```bash
npx calllint scan .cursor/mcp.json
```

## What "stable" means here

The **CLI contract, verdict semantics (`SAFE`/`REVIEW`/`BLOCK`/`UNKNOWN`), report
schema v0, release chain, and CI integration are stable.** It does **not** mean any
scanned tool is proven safe — CallLint is a static, offline, heuristic pre-flight
scanner that never executes the server it judges (see `SECURITY.md` /
`LIMITATIONS.md`).

## Headline since the preview line

- **RC-BLK-01 fixed** (shipped in `0.3.0-rc.1`): an MCP server config whose runtime
  the parser could not recognize — a nested `mcpServers.<name>.server.url`, a
  typo'd key hiding a remote URL, or an empty server object — previously resolved
  to `SAFE`. It now resolves to `UNKNOWN`. `SAFE` requires a positively recognized
  source; "we examined nothing" is never reported as "no blockers observed."
  Found during the rc.0 feedback window by scanning real third-party public-repo
  configs. See [ADR 0010](../adr/0010-unknown-runtime-fails-to-unknown.md).
  - **Scope (precise):** this resolves the *verdict-engine* fail-open. The parser
    does not yet *positively* recognize a nested/aliased `server.url` as a remote;
    such shapes reach `UNKNOWN` via the unknown-source path (the safe direction).
    Regression-locked by a golden fixture, corpus case `C031`
    (`thisCaseMustNeverBeSafe`), and unit tests.
- **R2.1 corpus** — 31 calibrated cases (21 real-public or redacted-real
  snapshots), dangerous false-SAFE = 0, UNKNOWN ratio 12.9% (target < 15%),
  enforced by `corpus:test:r2-final`.
- **SARIF dogfood**, **Website V3** (agent-readable surface), and **Trusted
  Publishing** with build provenance — carried from the preview/rc line.

## Known limitations

- Static configuration scanner only — does not execute target MCP servers; does
  not prove runtime safety (a clean run is necessary, not sufficient).
- The parser does not yet positively recognize nested/aliased `server.url` as a
  remote (it fails closed to `UNKNOWN`); an unrecognized local `command` is treated
  as a known source (tracked as a non-blocking calibration item for the next corpus
  round).
- The R2.1 corpus meets its thresholds but does not yet represent the full MCP
  ecosystem; expansion continues.
- Verdicts are heuristic decision support, not a safety guarantee.
