# CallLint v0.3.0-preview.0

> Draft GitHub Release notes. Mark as **Pre-release**. Tag: `v0.3.0-preview.0`.

This is the first public preview of CallLint.

CallLint is a deterministic, offline-first CLI that scans MCP and agent-tool
configurations **before execution** and returns SAFE, REVIEW, BLOCK, or UNKNOWN
with evidence. It never executes, installs, or connects to the server it judges.

```bash
npx calllint scan .cursor/mcp.json
```

## Highlights

- Public npm preview: `calllint@0.3.0-preview.0` (preview dist-tag).
- Deterministic verdict path — no LLM in the decision path.
- Offline by default; `--online` is advisory and never upgrades a verdict.
- JSON, SARIF (GitHub Code Scanning), and HTML report output.
- Baseline / verify drift detection (rug-pull signal on pinned-version changes).
- R2.0 corpus gate — 10 calibrated seed cases with a `corpus:test` release gate.
- Trusted Publishing release workflow (OIDC + provenance; no long-lived token).
- calllint.com public website.

## Known limitations

- Static configuration scanner only — does not execute target MCP servers.
- Does not prove runtime safety; a clean run is necessary, not sufficient.
- Corpus is seed-stage (R2.1 real/redacted expansion is pending), not an
  ecosystem-wide benchmark.
- Verdicts are heuristic decision support, not a safety guarantee.
- Pre-1.0 preview.
